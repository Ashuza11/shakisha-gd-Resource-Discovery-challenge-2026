# Submission Guide (Participants)

## Deadline package

Submit all of the following:

1. Repository URL
2. Streamlit run command and dependencies
3. README with setup and usage
4. Short architecture/logic note
5. Data usage and provenance note
6. Demo script/slides
7. One policy/advocacy scenario walkthrough

## Required content in README

- project objective
- user persona
- key workflows
- setup steps
- limitations
- next steps

## Provenance requirement

For key resources validated during event, include:

- source URL
- access time
- access status
- notes on restrictions or issues

## Naming convention

- Team repo: `team-<name>-gdrh-2026`
- Optional zip: `team-<name>-submission.zip`

## Final check before submission

- [ ] App runs from clean environment
- [ ] Required docs are present
- [ ] Source attribution included
- [ ] Demo flow rehearsed
